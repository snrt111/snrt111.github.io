Windows安装OpenClaw保姆级教程

发表于 2026-03-24 更新于 2026-03-28 分类于 AI 与人工智能

前言

OpenClaw（俗称”龙虾”）是一款面向本地运行、轻量高效的开源AI工具，提供了强大的大模型部署和推理能力。本教程将手把手教您在Windows系统上安装和配置OpenClaw，即使是电脑小白也能轻松完成。

系统要求检查

1. 确认Windows版本

操作步骤：

  1. 按下键盘上的 Win + R 键（同时按住Windows键和R键）
  2. 在弹出的”运行”窗口中输入 winver
  3. 点击”确定”按钮

查看结果：

  • 如果显示 Windows 10 或 Windows 11，且是64位系统，可以继续安装
  • 如果显示Windows 7/8，需要先升级系统

2. 确认系统位数

操作步骤：

  1. 右键点击”此电脑”（或”我的电脑”）
  2. 选择”属性”
  3. 查看”系统类型”

要求： 必须显示 64位操作系统

3. 检查磁盘空间

操作步骤：

  1. 打开”此电脑”
  2. 查看C盘可用空间

要求： C盘至少预留 2GB 可用空间（建议5GB以上）

4. 检查内存

操作步骤：

  1. 右键点击”此电脑”
  2. 选择”属性”
  3. 查看”机带RAM”

要求：

  • 最低 4GB 内存
  • 推荐 8GB 或以上（运行大模型更流畅）

  3. 点击启动

  • 显示 Server started at http://localhost:8080

3. 完成首次配置

3.1 设置管理员账号

操作步骤：

  1. 在”管理员账号”页面，输入以下信息：

    • 用户名：输入您想要的登录名（如admin）
    • 密码：设置登录密码（至少6位）
    • 确认密码：再次输入相同密码

  2. 点击”下一步”或”创建账号”

重要提示：

  • 请牢记用户名和密码
  • 建议将密码记录在安全的地方

3.2 选择界面语言

操作步骤：

  1. 在语言选择页面，点击下拉菜单
  2. 选择 简体中文
  3. 点击”下一步”

3.3 配置存储路径

操作步骤：

  1. 保持默认路径（推荐）：C:\Users\你的用户名\.openclaw
  2. 或者点击”浏览”选择其他位置
  3. 确保选择的磁盘有足够的空间（至少10GB）
  4. 点击”下一步”

3.4 完成配置

  1. 查看配置摘要
  2. 确认无误后点击”完成”
  3. 等待系统初始化（约1-2分钟）

下载和使用AI模型

1. 进入模型管理页面

操作步骤：

  1. 在左侧菜单栏中找到 模型管理（图标通常是🤖或📦）
  2. 点击进入

2. 浏览可用模型

页面说明：

  • 官方模型：OpenClaw推荐的常用模型
  • 社区模型：用户分享的模型
  • 已安装：已下载到本地的模型

3. 下载模型（以Llama为例）

操作步骤：

  1. 在模型列表中找到 Llama 2 或 Llama 3
  2. 点击模型名称查看详情
  3. 根据您的显存选择合适版本：
    • 7B版本：适合4GB显存，约4GB大小
    • 13B版本：适合8GB显存，约8GB大小
    • 70B版本：需要大显存，约40GB大小
  4. 点击”下载”按钮

下载过程：

  • 显示下载进度条
  • 下载时间根据网络速度，约10-30分钟
  • 可以最小化窗口，后台继续下载

4. 启动模型进行对话

操作步骤：

  1. 下载完成后，在”已安装”标签页找到该模型
  2. 点击”启动”按钮
  3. 等待模型加载（首次加载需要1-3分钟）
  4. 加载完成后，点击”开始对话”

5. 进行AI对话

操作步骤：

  1. 在对话框中输入您想问的问题
  2. 按回车键或点击”发送”按钮
  3. 等待AI回复

参数说明：

  • Temperature（温度）：控制回答的创造性

    • 值越低（如0.2）：回答更保守、准确

常见问题解决

问题1：安装时提示”无法连接到服务器”

原因： 网络连接问题或防火墙阻止

解决方法：

  1. 检查网络连接是否正常
  2. 暂时关闭Windows Defender防火墙：
    • 打开”设置” → “更新和安全” → “Windows安全中心”
    • 点击”防火墙和网络保护”
    • 暂时关闭”专用网络”防火墙
  3. 重新运行安装命令
  4. 安装完成后记得重新开启防火墙

问题3：启动后浏览器显示”无法访问此网站”

原因： OpenClaw服务未正常启动

解决方法：

  1. 检查PowerShell窗口是否还在运行
  2. 查看窗口中是否有错误信息
  3. 尝试重新启动：
    • 关闭PowerShell窗口
    • 重新打开PowerShell（管理员）
    • 输入 openclaw 启动
  4. 如果还是不行，尝试重启电脑后再启动

问题4：模型下载速度慢或中断

原因： 网络不稳定或模型源在国外

解决方法：

  1. 使用国内镜像源：

    • 在模型管理页面，找到”设置”
    • 开启”使用国内镜像”
    • 重新下载

  2. 使用下载工具：

    • 在模型详情页，找到”手动下载”链接
    • 使用迅雷等下载工具下载
    • 下载完成后，在OpenClaw中选择”导入本地模型”

问题5：模型加载时电脑变得很卡

原因： 电脑配置不足，模型占用资源过多

解决方法：

  1. 关闭其他程序：

    • 关闭不必要的浏览器标签页
    • 退出游戏、视频播放器等占用资源的程序

  2. 使用更小的模型：

    • 卸载大模型
    • 下载7B或更小参数的模型

  3. 降低模型参数：

    • 在模型设置中，减少”上下文长度”
    • 降低”批处理大小”
